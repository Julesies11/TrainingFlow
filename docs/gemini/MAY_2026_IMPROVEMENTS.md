# May 2026 Features & Fixes Implementation Report

## Overview
This report documents the series of enhancements, UI fixes, and critical data bugs resolved in the TrainingFlow application related to the Workout Library, Template Builder, and Calendar interactions.

## 1. Library Drawer Enhancement
**Objective:** Allow users to create new workout templates directly from the slide-out Library Drawer without navigating away to the main Library page.
- **Changes:** Added a "+" button to the header of the `LibraryDrawer` component (`src/pages/training/calendar/components/library-drawer.tsx`).
- **Functionality:** Clicking the button opens the existing `LibraryTemplateDialog`, allowing users to define a new template. Upon saving, the template is added to the library using the existing mutation hooks.
- **UI Refinement:** Added right-padding (`pr-14`) to the drawer header to prevent the new button from overlapping with the absolute-positioned Sheet close button.
- **Verification:** Unit tests were added in `src/pages/training/calendar/components/__tests__/library-drawer.test.tsx` to ensure the button renders and triggers the dialog.

## 2. Template Builder Data Integrity (Recurrence Bug)
**Objective:** Fix a critical bug where repeated workouts added from the library into a training plan would all collapse onto a single day.
- **Root Cause:** The `TemplateBuilderDialog`'s save logic relied on stale `weekNumber` and `dayOfWeek` properties on the workout objects, ignoring the correctly incremented `date` fields generated by the recurring logic.
- **Changes:**
  - `src/pages/account/templates/components/template-builder-dialog.tsx`: Updated `handleSaveWorkout` and `handleSaveBulk` to *always* recalculate the `weekNumber` and `dayOfWeek` from the workout's `date` using the `getCoordinatesFromDate` helper.
  - `src/pages/training/calendar/components/workout-dialog.tsx`: Enhanced the initialization logic so that when a library workout is added in template mode, it immediately calculates its week/day coordinates based on the drop date, rather than defaulting to Week 1.

## 3. Template Builder Save Workflow
**Objective:** Keep the Template Builder page open after successfully saving a plan, enabling continuous editing.
- **Changes:** Removed the `onClose()` calls from the `onSuccess` callbacks in the `handleSave` function of `TemplateBuilderDialog`.
- **State Sync:** Refactored `initialStateString` from a static `useMemo` to a dynamic `useState`. It now updates to the newly saved data upon success. This correctly resets the `isDirty` flag, preventing false "unsaved changes" warnings.
- **Creation Flow:** For newly created plans, the generated ID is now captured and mapped back into the form data, seamlessly switching the UI from "create" to "edit" mode.

## 4. UI Layering & Visibility (Z-Index Overhaul)
**Objective:** Fix the issue where the `WorkoutDialog` appeared *behind* the `LibraryDrawer`, and improve background calendar visibility.
- **Z-Index Hierarchy Fix:**
  - `Sheet` (Drawers): `z-[110]`
  - `Dialog` (Modals): Increased to `z-[150]`.
  - `Select`, `DropdownMenu`, `Popover`: Increased to `z-[160]` so they pop out of dialogs.
  - `Tooltip`: Increased to `z-[200]`.
- **Transparency Improvements:**
  - Reduced `DialogOverlay` to `bg-black/20` and `blur(2px)`.
  - Reduced `SheetOverlay` to `bg-black/10` and `blur(1px)`.
  - **Result:** Users can now clearly see the background calendar when interacting with the library and adding workouts.

## 5. Workout Dialog Day Selector
**Objective:** Replace the numeric "day of week" input with a user-friendly dropdown when duplicating workouts in template mode.
- **Changes:** Replaced the HTML numeric `<input>` with the ReUI `<Select>` component in `WorkoutDialog`. Displays labels like "Monday" to "Sunday" while mapping them to database-compatible values (`1-7`).

## 6. CSV Import Boolean Parsing
**Objective:** Fix an issue where the `isKeyWorkout` flag always evaluated to `true` when imported from CSV.
- **Changes:** Replaced `z.coerce.boolean()` with a `z.preprocess()` function in `ImportWorkoutSchema` (`src/services/training/import.utils.ts`). It now correctly interprets CSV string values like `"false"`, `"0"`, `"true"`, and `"1"`.
- **Refactoring:** Consolidated the JSON and CSV processing paths in `parseImportData` to ensure consistent validation.
- **Database Update:** Created `supabase/migrations/2026051301_fix_template_key_workouts.sql` to permanently remove the `is_key_workout` flag from non-key template sessions (Mondays, Fridays, and Gym/Strength).
